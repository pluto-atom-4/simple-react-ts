## Exercise Coding Exercises for React Frontend

Many sites provide starter code or templates in online editors like CodePen, CodeSandbox, or their own IDEs, allowing you to complete and test implementations.

Many include partial code (e.g., stubs for components/hooks) to complete. Focus on "Design Front-End Component" or JS-based UI tasks.

### Practice Routine

- Timed Sessions: Allocate 45-60 min per problem (match screen length). Solve 3-5 problems/day from the list (e.g., Todo List with Filtering, Infinite Scroll). Think aloud: "I'll use useState for list; edge case: empty input."
- Focus Areas:Hooks/Components: Build UIs like Accordion or Counter with Undo (test re-renders via console.log).
    - Performance: Add memoization to lists; explain Big-O for renders.
    - Integration: Mock GraphQL fetches with useEffect; handle loading/errors.
    - Debugging: Intentionally add bugs (e.g., infinite loop) and fix while verbalizing.

### Practice Tips
- Start with partial code: Copy starters to CodeSandbox/Replit; add blocks for state, events, rendering.
- Focus on hooks, components, optimization — common in Opendoor screens.
